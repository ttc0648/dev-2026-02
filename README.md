# 任务数据面板

## 项目简介

这是一个使用 JavaScript 完成 API 交互与数据渲染的任务面板页面。

## 我完成的功能

- ✅ 页面加载时自动获取任务列表
- ✅ 将任务数据渲染到页面
- ✅ 点击刷新按钮重新加载任务
- ✅ 提交新任务并更新列表
- ✅ 按状态筛选任务（全部/待开始/进行中/已完成）
- ✅ 请求失败时显示错误提示

## 我的实现思路

### 1. 页面加载
我在 `DOMContentLoaded` 事件中调用 `fetchTasks` 函数，页面打开后自动请求任务列表接口。

### 2. 获取和渲染数据
我使用 `fetch` 发送 GET 请求到 `/api/tasks`，把返回的 `data` 数组保存到 `currentTasks` 变量中，然后遍历数组动态创建 HTML 元素显示在页面上。

### 3. 提交任务
我监听了表单的 `submit` 事件，获取用户输入的任务名称、负责人和状态，用 POST 请求发送到服务器。成功后清空表单并重新获取任务列表。

### 4. 状态筛选
我把所有任务保存在 `currentTasks` 数组中，当下拉框切换时，用 `filter` 方法筛选出对应状态的任务，然后重新渲染。

### 5. 错误处理
我用 `try...catch` 捕获请求错误，并写了一个 `showMessage` 函数在页面上显示成功或失败的提示信息。

## 遇到的问题和解决方法

**问题1**：一开始筛选功能不生效
**解决**：我发现是筛选器的值（中文）和 API 返回的状态值（英文）不匹配，于是写了状态映射来解决。

**问题2**：刷新后筛选状态会重置
**解决**：我在刷新任务后判断当前筛选器的值，如果不是"全部"就自动应用筛选。
